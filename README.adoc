= DynBench Frontend

DynBench is a web application for generating robust benchmark records by transforming question-query pairs using AI models.

---

toc::[]

---

== Features

* Generate new benchmark items from question-query pairs (natural-language questions with SPARQL queries)
* Support for generating new questions-query pairs from questions written in multiple languages (20 languages)
* Adjustable comparability of generated questions through the use of entities for generated questions (importance of entities in generated questions in comparison to is chosen to ensure difficulty to be "easy", "similar", "hard", or "random")
* Provide random, multilingual samples from benchmark datasets for easier tool exploration
* Feedback collection for computed transformation steps
* Benchmark file upload mode: process a complete benchmark file pair by pair and export the transformed benchmark in the exact format of the uploaded file

== Benchmark file upload mode

Switch the input mode from "Single question-query pair" to "Benchmark file upload" to process a complete benchmark file. Every question-query pair is sent iteratively to the DynBench backend with the settings from the sidebar (model, entity difficulty, target language). For each pair a result block shows the given question and SPARQL query next to the computed pair; recognized named entities in the questions are highlighted and a mouse-over overlay shows the human-readable label and the knowledge-graph resource URL. Each computed pair can be rated (👍/👎), which is submitted to the regular feedback endpoint.

The number of processed pairs is user-configurable (default: 10, counted from the beginning of the file) because every pair is one LLM round-trip. The computed pairs can be downloaded as a new benchmark file in the *exact same format* as the uploaded file.

The file format is detected automatically (see `benchmark_formats.py`). Supported formats:

* QALD-style JSON (QALD-9, QALD-9-plus, QALD-10; also DBLP-QuAD, SciQA, VQuAnDa and BESTIARY variants)
* QALD XML (older QALD editions, e.g. QALD-4 biomedical)
* LC-QuAD 1.0 and LC-QuAD 2.0
* RuBQ 1.0/2.0
* GrailQA
* WebQuestionsSP
* JSON lists with `question` + `sparql` keys (KQA Pro, ComplexWebQuestions, CFQ exports, …)
* Flat DynBench JSON (the bundled DynQALD/DynRuBQ sample format)
* CSV/TSV with named question and SPARQL columns (e.g. Spider4SPARQL)
* Single SPARQL files (`.rq`/`.sparql`) with the question in leading `#` comments (Bio-SODA benchmark style)
* SIB sparql-examples Turtle files (`rdfs:comment` + `sh:select`)

=== Programmatic access (REST API)

The same batch processing is available to programs via `POST /api/transform-benchmark` (interactive Swagger documentation at `GET /api/transform-benchmark`, machine-readable spec at `/api/openapi.json`). Upload the benchmark file as multipart/form-data (field `file`) or as the raw request body with a `filename` parameter for format detection:

```bash
# per-pair JSON report for the first 10 pairs
curl -X POST 'https://<host>/api/transform-benchmark?model=gpt-4o&limit=10' \
  -F 'file=@qald-9-test.json'

# transformed benchmark in the exact format of the upload, all pairs
curl -X POST 'https://<host>/api/transform-benchmark?model=gpt-4o&limit=0&response=file' \
  -F 'file=@qald-9-test.json' -o qald-9-test-transformed.json
```

Parameters: `model` (required, see `GET /api/models`), `complexity` (`easy|normal|hard|random`, default `normal`), `language` (ISO 639-1, default `en`), `limit` (pairs from the beginning of the file, default `10`, `0` = all), `response` (`json` = per-pair report incl. entity replacements and per-pair errors, `file` = the transformed benchmark in the upload's format). Note that every pair is one LLM round-trip, so large `limit` values lead to long-running requests.

== Prerequisites

* Python 3.10+
* Docker (optional, for containerized deployment)

== Installation

### Clone the repository

```bash
git clone <repository-url>
cd DynBench-Frontend
```

=== Install dependencies

Using pip:

```bash
pip install -r requirements.txt
```

Or using pipenv:

```bash
pipenv install
```

== Configuration

The application requires environment variables to be set. Create a `.env` file in the project root:

```env
DYNBENCH=<Path to LLM transform endpoint>
MODEL=<Your model>
```

=== Environment Variables

[cols="1,2,2"]
|===

| Variable
| Description
| Default

| `DYNBENCH`
| URL of the DynBench backend service
| -

| `MODEL`
| AI model to use for transformations
| -

| `GITHUB_REPO`
| GitHub repository URL for the fork ribbon
| https://github.com/WSE-research/DynBench-Frontend.git

|===

== Running the Application

=== Quick start (Linux/macOS)

[source,bash]
----
./start.sh              # create .venv, install dependencies, start the app
PORT=8600 ./start.sh    # start on a custom port
./start.sh --dev        # additionally install the dev tooling (pytest, ruff)
----

The script checks that `MODEL` and `DYNBENCH` are configured (environment or `.env`, see <<configuration>>), reinstalls dependencies only when the requirements files changed, and launches the app through `run.py` so that the REST API (`/api/transform`, `/api/transform-benchmark`) and the `/health` endpoint are registered alongside the Streamlit UI at `http://localhost:8501`.

=== Running with Python (manual)

1. Ensure you have installed the dependencies:
+
[source,bash]
----
pip install -r requirements.txt
----
2. Set up your `.env` file (see <<configuration>>)
3. Run the application:
+
[source,bash]
----
python run.py           # UI + REST API + /health (recommended)
streamlit run app.py    # UI only — the /api/* endpoints are NOT registered
----
4. The application will open in your browser at `http://localhost:8501`


=== Running with Docker

1. Build the Docker image:
+
[source,bash]
----
docker build -t dynbench-frontend .
----
2. Run the container with environment variables:
+
[source,bash]
----
docker run -p 8501:8501 \
       -e DYNBENCH=http://your-backend-url:port/transform \
       -e MODEL=mistral-small \
       dynbench-frontend
----

3. The application will be available at `http://localhost:8501`

=== Running with Docker Compose (optional)

Create a `docker-compose.yml`:

```yaml
version: '3.8'
services:
  dynbench-frontend:
    build: .
    ports:
      - "8501:8501"
    environment:
      - DYNBENCH=http://your-backend-url:port/transform
      - MODEL=mistral-small
    restart: unless-stopped
```

Then run:

```bash
docker-compose up -d
```

== Usage

1. **Select difficulty**: Choose from easy, normal, hard, or random
2. **Select language**: Pick from 20 supported languages
3. **Enter question and query**: Provide a natural language question and its corresponding SPARQL query
4. **Generate**: Click the "Generate" button to transform the pair
5. **Review results**: Check the transformed question and query
6. **Provide feedback**: Use the OK/Wrong buttons to give feedback on each result

== Benchmark Datasets

The application includes sample datasets:
- `benchmarks/DynQALD.json` - Question Answering Benchmark
- `benchmarks/DynRuBQ.json` - Russian/English Benchmark
